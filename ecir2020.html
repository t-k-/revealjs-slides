Path-Based Inverted Index

  • Use prefix leaf-root paths from operator tree
  • Index (by ExpID) and retrieve formulas on an inverted index
  • "Recover" structure by grouping root-end node IDs
  • Calculate similarity score $\gamma(\pi)$

Dynamic Pruning

  • Dynamic pruning speeds up query by skipping documents unlikely to be ranked in top-K results
  • Assume the minimal score of top K candidates at a given point is threshold $\theta$. Any hit cannot appear in the final top-K if its upperbound $\le \theta$
  • In MaxScore [1-3], if a set of posting lists have upperbound $\le \theta$, then the associated document can be skipped. They are called non-requirement set.

Model Path Matching

The number of matched leaves for $m \in T _ { q } , n \in T _ { d }$ is $$ \sum _ { t \in \mathfrak { T } ( m ) } \operatorname { min } ( | Q _ { t } ^ { ( m ) } | , | D _ { t } ^ { ( n ) } | ) $$ where $\mathfrak { T }(m)$ is the set of tokenized paths attached to node $m$.

Model Path Matching (Example)

If $m, n$ are the highlighted ADD nodes, then $\mathfrak { T } ( m ) = $ { /VAR/TIMES/ADD, /VAR/ADD }.
The maximum matched number of paths is $\color{red}{\min(2, 2)} + \color{blue}{\min(2, 1)} = 3$.

Path Posting List Upperbound

  • Simplify the similarity measurement, only score by the width of the widest common tree $$ w^*_{Q, D} = \max_{m\in T_q, n \in T_d} \sum _ { t \in \mathfrak { T } ( m ) } \operatorname { min } ( | Q _ { t } ^ { ( m ) } | , | D _ { t } ^ { ( n ) } | ) $$
  • Let $w_{m, t} = |Q_t^{(m)}|$, then $w_{m, t} \ge \min(|Q_t^{(m)}|, |D_t^{(n)}|)$
  • $w_{m, t}$ can be viewed as a partial upperbound of our desired score
  • $w_{m, t}$ can be pre-computed since the inequality does not depend on document

Required Set Selection Strategies

  1. MaxRef: Assign upperbounds for each posting list $t$ explicitly

    $$ \text{MaxRef}_t = \max_m \; w_{m, t} $$
  2. GBP-NUM: Maximize number of posting lists in non-required set:

    $$ \text{max} \qquad 1^\top \mathbf{x} \qquad s.t. \quad \mathbf{W} \mathbf{x} \le \theta \\ $$
  3. GBP-LEN: Maximize total length of posting lists in non-required set:

    $$ \text{max} \qquad [L_1, L_2, ...] \;\mathbf{x} \qquad s.t. \quad \mathbf{W} \mathbf{x} \le \theta \\ $$
(*) Since binary programming problem is hard, we greedily follow one branch of the sub-problems to obtain a feasible (sub-optimal) solution.

Final Score

The "structure score" is simply the width of maximum matched common tree.

$$ w^*_{Q, D} = \max_{m\in T_q, n \in T_d} \sum _ { t \in \mathfrak { T } ( m ) } \operatorname { min } ( | Q _ { t } ^ { ( m ) } | , | D _ { t } ^ { ( n ) } | ) $$

The overall score [1] further captures symbol similarity (e.g., to discern
$E=mc^2$ vs. $y=ax^2$) and penalizes large formulas.

$$ u ( w ) = \frac { w } { | \operatorname { leaves } ( T _ { q } ) | + w } \left[ ( 1 - \eta ) + \eta \frac { 1 } { \operatorname { log } ( 1 + n _ { d } ) } \right] $$
where $\eta = 0.05, n_d = 1$.
[1]: Zhong, Wei, and Richard Zanibbi. "Structural similarity search for formulas using leaf-root paths in operator subtrees." ECIR 2019

Evaluation

  • 2016 NTCIR-12 Wiki dataset [1] of 590,000 formulas from English Wikipedia, including 20 non-wildcards queries.
  • Math StackExchange (MSE) corpus [2] with ~1 million documents from mathematics Q&A, this corpus is generated by our own.
  • Bpref [3] scores are compared among systems for effectiveness (Wiki). Efficiency is evaluated by query run times statistics of 5 independent runs (MSE and Wiki).

Runtimes (ms) for Pruning Strategies

  • For all average runtimes, GBP-LEN consistently outperform others
  • GBP strategies have tighter upperbound than MaxRef, and GBP-LEN models the pruning possibility better than GBP-NUM
  • Comparing to existing systems (*), we also outperform them in both median and max values.
(*) We are comparing our baseline model and Tangent system from NTCIR-12. MCAT is not included here because it reportedly has a median query execution time around 25 seconds. Text-based systems in NTCIR-12 such as the MIaS system from Masaryk University is also excluded here as it is 2~5 times worse effective than structure-based retrieval systems.

Conclusion / Future Work

  • We proposed three dynamic pruning strategies for math formula retrieval
  • We associated structure representation to required set selection
  • We have made highly effective substructure retrieval more efficient
  • In the future, we consider to incorporate TF-IDF into our scoring function
